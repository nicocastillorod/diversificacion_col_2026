Identificando la heterogeneidad en tasas de diversificación

Tutorial y script creado por Juan Diego Rodríguez-Rodríguez.

Actualizado en Julio 2026.

En este tutorial vamos a aprender a utilizar tres métodos diferentes para estimar la heterogeneidad en las tasas de diversificación: Lineage Specific Birth Death Shift Model (LSBDS) (Hönha, 2026); Cladogenetic Diversification rate Shift (ClaDS) model . Estos métodos se caracterizan por estimar las tasas de diversificación para cada una de las ramas de la filogenia estudiada. Existen una gran cantidad de métodos utilizados para este fin, pero difieren en sus supuestos, algoritmos y en los resultados que arrojan. Por lo que es indispensable ser conciente de las limitaciones de cada método y la inciertidumbre asociada al mismo. Lo recomendable es utliizar varios métodos y comparar sus resultados, evitando reportar solo estimadores puntuales.

Filogenia a utilizar

En este tutorial vamos a análizar la filogena de la familia Liolaemidae publicada por Esquerré et al (2019) https://doi.org/10.1111/evo.13657, una familia de lagartos que habitan en la parte sur de Sudamérica, cuya diversidad de especies está principalmente dada por el género Liolaemus y muy ligada a los hábitats andinos de alta montaña.

Vamos a crear tres carpetas para el tutorial: data output y scripts, el unico archivo que vamos a usar en este tutorial se llama Liolaemidae_2022.nex, el cual corresponde al árbol de máxima credibilidad (MCC).

Lineage Specific Birth Death Shift Model

Primero vamos a ajustar un modelo LSBDS, el cual discretiza las tasas de diversificación ajustando una distribución continua en n intervamos (Igual que Yang, 2004) para evitar tener que calcularlos en una distribución continua (Lo cual es computacionalmente imposible) para despues asignar estas tasas a cada rama a tráves de un mapeo estocastico (SCM).

Este modelo es implementado en el Software RevBayes https://revbayes.github.io/, el cual es un programa que ofrece mucha flexibilidad en la construcción de los modelos, ya que exige especificar cada uno de los parametros del mismo.

Este programa tiene dos variables principales, la version “base”, que se ejecuta como rb, y la versión “multicore”, que se ejecuta como rb-mpi y permite que cada uno de los nucleos ejecute una cadena del análisis.

##Filogenia
phy <- readTrees( "./data/Liolaemidae_2022.nex")[1]

##Información de la filogenia
taxa <- phy.taxa()
root <- phy.rootAge()
tree_length <- phy.treeLength()

##Movimientos y monitores
moves = VectorMoves()
monitors = VectorMonitors()

##Parametros globales
NUM_RATES = 10 ##Vamos a dividir el continuo de las tasas de diversificación en 10 categorías discretas
EXP_NUM_SHIFTS = 10 ##Esperamos encontrar más o menos 10 cambios en las tasas de diversificación 



##Probabilidad de muestreo, como no tenemos información de todas las especies actuales (algunas aún no están descritas) es necesario especificar la probabilidad de que una especie sea incluida en la filogenia.
NUM_TOTAL_SPECIES = 353
rho<-phy.ntips() / NUM_TOTAL_SPECIES 


###Prior on specieation and extintion rates 
rate_mean <- (NUM_TOTAL_SPECIES-2) / tree_length

H = 0.587405 #Desviación estandar de las tasas, este valor es un prior razonable para poder representar diferencias de 3 a 100 veces en las tasas de diversificación 
#Honha et al. (2017) 
rate_sd <- H
##Distribución exponencial para la desviación estandar de las tasas de diversificación
rate_sd ~ dnExponential(1.0 / H)
moves.append( mvScale(rate_sd, lambda =1, tune = true, weight =2.0) )

##Prior con distribución LogNormal discretizada para las tasas de especiación
speciation_mean ~ dnUniform( 0.0, 100 )
speciation_mean.setValue( rate_mean )
moves.append( mvScale(speciation_mean, lambda = 1, tune = true, weight = 2.0) )
speciation := fnDiscretizeDistribution( dnLognormal(ln(speciation_mean),rate_sd),NUM_RATES)

##Prior  uniforme para las tasas de extinción
extinction_mean ~ dnUniform ( 0.0, 100 )
extinction_mean.setValue( rate_mean / 2.0 ) ##La mitad de las tasas de especiación
moves.append( mvScale(extinction_mean, lambda = 1, tune = true, weight = 2.0))
##Asumir que las tasas de extinción son constantes en cada una de las categorias (una tasa de extinción global)
extinction := rep(extinction_mean,NUM_RATES)

### Prior con distribución uniforme para la tasa de cambio entre regimenes de diversificación
shift_rate ~ dnUniform(0.0, 100.0/tree_length)
shift_rate.setValue(2.0/tree_length)
moves.append( mvScale(shift_rate, lambda=1, tune=true, weight=2.0) )

##Uniform dist for rate category at the root
##Prior con distribución uniforme para cada categoría 
rate_cat_probs <- simplex( rep(1,NUM_RATES))

##Variable para el árbol producto del proceso de nacimiento y muerte 
timetree ~ dnCDBDP (rootAge = root,
                    speciationRates = speciation,
                    extinctionRates = extinction,
                    Q = fnJC(NUM_RATES),
                    delta = shift_rate,
                    pi = rate_cat_probs,
                    rho = rho,
                    condition = "time")

###Establecer el árbol como observado
timetree.clamp(phy)


##Configurar modelo
mymodel = model(speciation) 

#Configurar los monitores, uno para los diagnosticos del modelo y otro con los resultados del SCM
monitors.append( mnScreen(printgen=1000, shift_rate, speciation_mean) )
monitors.append( mnStochasticBranchRate(cdbdp=timetree, printgen=100, filename="output/liolaemidae_LSBDS_SCM.log") )
monitors.append( mnModel(filename="output/liolaemidae_LSBDS_model.log") )

##MCMC
mymcmc = mcmc(mymodel, monitors, moves, nruns=16, combine="mixed")

##Al ser cadenas que pueden ser largas (varios días/semanas) idealmente hay que generar un archivo de punto de guardado para poder reanudar en análisis si llega a interrumpirse

#mymcmc.initializeFromCheckpoint("output/liolaemidae_LSBDS_model.state")

stopping_rules[1] = srMinESS(250, file = "output/liolaemidae_LSBDS_model.log", freq = 10000)

mymcmc.run(rules = stopping_rules, checkpointInterval = 1000, checkpointFile = "output/liolaemidae_LSBDS_model.state")

##Correr el MCMC con generaciones fijas
mymcmc.run(generations = 50000, tuningInterval=200, checkpointFile = "output/liolaemidae_LSBDS_model.state")


##QUIT
q()

matchNodes = function(phy) {

  # get some useful info
  num_tips = length(phy$tip.label)
  num_nodes = phy$Nnode
  tip_indexes = 1:num_tips
  node_indexes = num_tips + num_nodes:1

  node_map = data.frame(R=1:(num_tips + num_nodes), Rev=NA, visits=0)
  current_node = phy$Nnode + 2
  k = 1
  t = 1

  while(TRUE) {

    if ( current_node <= num_tips ) {
      node_map$Rev[node_map$R == current_node] = t
      current_node = phy$edge[phy$edge[,2] == current_node,1]
      t = t + 1
    } else {

      if ( node_map$visits[node_map$R == current_node] == 0 ) {
        node_map$Rev[node_map$R == current_node] = node_indexes[k]
        k = k + 1
      }
      node_map$visits[node_map$R == current_node] = node_map$visits[node_map$R == current_node] + 1

      if ( node_map$visits[node_map$R == current_node] == 1 ) {
        # go right
        current_node = phy$edge[phy$edge[,1] == current_node,2][2]
      } else if ( node_map$visits[node_map$R == current_node] == 2 ) {
        # go left
        current_node = phy$edge[phy$edge[,1] == current_node,2][1]
      } else if ( node_map$visits[node_map$R == current_node] == 3 ) {
        # go down
        if (current_node == num_tips + 1) {
          break
        } else {
          current_node = phy$edge[phy$edge[,2] == current_node,1]
        }
      }
    }

  }

  return(node_map[,1:2])

}

addLegend = function(tree, bins, colors, width=0.1, height=0.4, lwd=1, title="posterior probability", ...) {

  lastPP <- get("last_plot.phylo", envir = .PlotPhyloEnv)

  x_left   = width
  x_right  = width + width * lastPP$x.lim[2]
  y_bottom = bins[-length(bins)] * height * length(tree$tip.label)
  y_top    = bins[-1] * height * length(tree$tip.label)

  ticks = pretty(bins)
  ticks = ticks[ticks > min(bins)]
  ticks = ticks[ticks < max(bins)]
  y_tick = ticks * height * length(tree$tip.label)
  if(lwd > 0) {
    segments(x0=x_right, x1=x_right + 0.01 * abs(diff(lastPP$x.lim)), y0=y_tick, lwd=lwd, ...)
  }
  text(x=x_right + 0.02 * abs(diff(lastPP$x.lim)), y=y_tick, label=ticks, adj=0, ...)
  rect(x_left, y_bottom, x_right, y_top, col=colors, border=colors)

  text(x_left - width / 1.5, mean(bins) * height * length(tree$tip.label), labels=title, srt=90, ...)
  # text(x=x_left, y=max(y_top) + 0.02 * length(tree$tip.label), labels=title, adj=0, ...)
  # points(x=x_left, y=max(y_top) + 0.05 * length(tree$tip.label))

}

Phylogenetic Estimation of Branch-Specific Shifts in the Tempo of Origination

Ahora, realizaremos un análisis utilizando el modelo PESTO, este método es implementado en el lenguaje de programación Julia y tiene la característica de no depender de cadenas de Markov (MCMC) por lo que es bastante eficiente computacionalmente (Puede procesar filogenias de más de mil terminales en un computador personal en un tiempo razonable), sin embargo solo resulta en un set de estimados puntuales, por lo que idealmente debe realizarse en una muestra de árboles de la distribución posterior (esto se puede realizar a traves de un loop).

import Pkg
Pkg.add(url="https://github.com/kopperud/Pesto.jl")

using Pesto

phy = readtree("./data/Liolaemidae_2022.nex")
sampling_fraction = 0.72 ##243 de las 353 especies reconocidas 
lio_divrates = SSEdata(phy, sampling_fraction)

model, rates = pesto(lio_divrates)


writenewick("./output/liolaemidae_pesto.tre", lio_divrates, rates)
Pkg.add("Makie")
Pkg.add("CairoMakie")
using Makie, CairoMakie
treeplot(lio_divrates,rates)

df = tip_rates(model, lio_divrates)

f = Figure(resolution = (600, 600))

axs = []
colors = [:blue, :red, :green, :orange]
for (i, rate) in enumerate([:lambda, :mu, :netdiv, :relext])
    ax = Axis(f[i,1],
        xgridvisible = false,
        ygridvisible = false,
        xlabel = string("Tip rate (", rate, ")"),
        ylabel = "Frequency")

    hist!(ax, df[!,rate], bins=30, color = colors[i])
end
rowgap!(f.layout, 5.0)
f

using Pkg
Pkg.add("CSV")
using CSV
CSV.write("./output/liolaemidae_pesto.csv", df)

Cladogenetic Diversification Shift model

Finalmente, vamos a hacer un análisis de ClaDS. Este método difiere de los dos anteriores debido a que busca detectar pequeños cambios en la tasa de diversificación en cada evento cladogenético en lugar de buscar regimenes discretos de diversificación.

Este modelo es implementado en Julia, aunque toma prestadas funciones de R a través del paquete RCall. Sin embargo, en el momento, por temas de compatibilidad es necesario tener la version de R 4.3.3.

Para windows

Descargar https://cran.r-project.org/bin/windows/base/old/4.3.3

Para MacOS

Para chips Apple Silicon descargar https://cran.r-project.org/bin/macosx/big-sur-arm64/base/R-4.3.3-arm64.pkg Para chips Intel descargar https://cran.r-project.org/bin/macosx/big-sur-x86_64/base/R-4.3.3-x86_64.pkg

Para Linux

Ejecutar el siguiente codigo en la terminal

wget https://cran.r-project.org/src/base/R-4/R-4.3.3.tar.gz
tar xzf R-4.3.3.tar.gz
